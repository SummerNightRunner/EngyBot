# Сценарии пользователя

## Точка входа

Основная точка входа — команда `/start`.

```text
/start
  -> профиль существует?
     -> нет:
        -> родной язык
        -> изучаемый язык
        -> уровень
        -> профиль сохранен
        -> главное меню
     -> да:
        -> главное меню
```

## Главное меню

```text
Главное меню
  -> Course
     -> Vocabulary
     -> Grammar
     -> Dialogues
  -> Practice
     -> Daily practice
     -> Mixed quiz
  -> Review
     -> Weak words
     -> Mistakes quiz
  -> Profile
  -> Progress
  -> Help
```

## Онбординг и регистрация

Онбординг собирает минимальный набор данных:

1. `source_language`;
2. `target_language`;
3. `level`.

После этого бот создает или обновляет профиль в `users`. Эта запись затем используется для:

- фильтрации контента;
- настройки bilingual UI;
- выбора тем и сложности учебных сессий.

## Course flow

### Vocabulary

1. пользователь открывает `Course`;
2. выбирает `Vocabulary`;
3. выбирает тему;
4. бот показывает карточки слов, отфильтрованные по уровню пользователя.

### Grammar

1. пользователь открывает `Course`;
2. выбирает `Grammar`;
3. открывает grammar unit;
4. бот показывает объяснение, шаблоны и примеры.

### Dialogues

1. пользователь открывает `Course`;
2. выбирает `Dialogues`;
3. открывает сценарий диалога;
4. бот показывает шаги сценария последовательно.

## Practice flow

### Mixed quiz

1. пользователь открывает `Practice`;
2. выбирает формат квиза;
3. выбирает тему;
4. бот собирает короткий набор вопросов;
5. после каждого ответа бот показывает мгновенную обратную связь;
6. в конце сессии выводится итог.

### Daily practice

1. пользователь открывает `Practice`;
2. выбирает `Daily practice`;
3. бот восстанавливает существующую сессию на текущую дату или создает новую;
4. пользователь проходит сессию до завершения.

Дневная практика закрепляется за текущим днем и не пересобирается при повторном входе.

## Review flow

1. пользователь проходит обычные quiz sessions;
2. бот обновляет статистику по отдельным словам;
3. слабые слова попадают в `Review`;
4. пользователь открывает список слабых слов или квиз по ошибкам.

## Progress flow

Раздел `Progress` сейчас показывает:

- количество квизов;
- точность;
- количество слабых слов;
- прогресс по темам;
- цель дня.

## Текущее ограничение

Сейчас маршрут остается section-based. Следующая целевая модель — unit-based:

```text
unit
  -> lesson
  -> vocabulary
  -> grammar
  -> dialogue
  -> practice
  -> review
```
