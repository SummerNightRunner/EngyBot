# Обзор проекта

## Назначение

EngyBot — это Telegram-бот для изучения языка. Проект построен как backend-first система: Telegram используется как канал взаимодействия, а логика обучения, фильтрация контента, сохранение прогресса и повторение ошибок реализованы на стороне приложения.

## Объем текущей реализации

Текущая версия включает:

- онбординг и сохранение профиля;
- словарные темы с фильтрацией по уровню;
- grammar units от `A1` до `C2`;
- короткие квиз-сессии;
- повторение на основе ошибок;
- дневную практику;
- прогресс и статистику.

Отдельного web-клиента, админ-панели и внешнего API в проекте нет.

## Основные разделы

### Course

Раздел `Course` содержит основной учебный материал:

- словарь;
- grammar units;
- мини-диалоги.

Темы словаря не привязаны жестко к одному уровню. Одна тема может использоваться на нескольких CEFR-уровнях, а состав слов внутри нее меняется в зависимости от уровня пользователя.

### Practice

Раздел `Practice` содержит активные учебные сессии:

- смешанный квиз;
- дневная практика.

Квизы сделаны короткими. Бот не проводит пользователя по всем словам темы за один раз, а формирует более компактную сессию и возвращает обратную связь после каждого ответа.

### Review

Раздел `Review` строится на пользовательской статистике:

- слабые слова;
- квиз по ошибкам.

Раздел повторения формируется не из статической темы, а из реальных ошибок пользователя.

### Progress

Раздел `Progress` показывает:

- количество попыток;
- точность;
- количество слабых слов;
- прогресс по темам;
- цель дня.

## Как работает регистрация

Регистрация в рамках проекта — это создание учебного профиля пользователя в таблице `users`.

Технический сценарий:

1. Пользователь отправляет `/start`.
2. Хендлер ищет запись по `telegram_id`.
3. Если записи нет, запускается FSM онбординга.
4. Пользователь выбирает:
   - `source_language`
   - `target_language`
   - `level`
5. После завершения онбординга бот записывает в базу:
   - `telegram_id`
   - `username`
   - `full_name`
   - `source_language`
   - `target_language`
   - `level`
   - `bilingual_ui`
6. После этого дальнейшие заходы идут уже через существующий профиль.

Этот профиль используется для подбора словаря, grammar units, интерфейсных подписей и учебных сценариев.

## Источники данных

Основные seed-файлы:

- `data/seed/word_sets.json`
- `data/seed/grammar_units.json`

Они содержат стартовый учебный контент, который загружается в базу.

## Работа с состоянием

Бот сочетает:

- постоянные данные в PostgreSQL;
- runtime-state через `aiogram` FSM.

Это используется для:

- онбординга;
- quiz flow;
- незавершенной дневной практики;
- промежуточных экранов обратной связи.

## Технические особенности текущей реализации

- профиль пользователя влияет и на доступный контент, и на UI;
- двуязычный режим можно включать и выключать в профиле;
- дневная практика хранится по дате и восстанавливается при повторном входе в тот же день;
- статистика по отдельным словам используется для повторения.

## Текущие ограничения

- курс пока построен по разделам, а не по полноценным units;
- grammar пока выступает скорее как контентный слой, чем как отдельный самостоятельный тренажер;
- большая часть хендлеров сосредоточена в `src/bot/handlers/start.py`;
- часть экранов все еще требует выравнивания текста и локализации.

## Следующий этап

Следующий крупный шаг — перейти от разделов к unit-based модели:

1. добавить `units`;
2. связать словарь, grammar, dialogues и practice с unit;
3. считать прогресс по units;
4. расширить практику по грамматике и упражнения по отдельным навыкам.
