# База данных

## Назначение

База данных хранит:

- профиль пользователя;
- учебный контент;
- результаты квизов;
- прогресс по отдельным словам;
- состояние daily practice.

Схема намеренно остается компактной и соответствует текущему MVP.

## Таблицы

### `users`

Профиль пользователя Telegram и его учебные настройки.

| Поле | Описание |
| --- | --- |
| `telegram_id` | Telegram user ID |
| `username` | Telegram username |
| `full_name` | отображаемое имя |
| `source_language` | родной язык |
| `target_language` | изучаемый язык |
| `level` | текущий CEFR-уровень |
| `bilingual_ui` | флаг bilingual interface |

Эта таблица определяет и доступный контент, и поведение интерфейса.

### `word_sets`

Словарные темы.

| Поле | Описание |
| --- | --- |
| `title` | название темы |
| `description` | краткое описание |
| `level` | рекомендуемый уровень входа |
| `is_active` | флаг доступности |

Уровень темы используется как нижняя граница доступа, а не как жесткое ограничение для всего содержимого темы.

### `words`

Слова и фразы внутри темы.

| Поле | Описание |
| --- | --- |
| `word_set_id` | родительская тема |
| `source_text` | текст на родном языке |
| `target_text` | текст на изучаемом языке |
| `level` | CEFR-уровень слова или фразы |
| `example` | пример употребления |

Уровень задается на уровне слова. Это позволяет одной теме развиваться через несколько уровней сложности.

### `training_attempts`

История завершенных квизов.

| Поле | Описание |
| --- | --- |
| `user_id` | ссылка на пользователя |
| `word_set_id` | ссылка на тему |
| `correct_answers` | число правильных ответов |
| `total_questions` | общее число вопросов |
| `created_at` | время завершения |

Эта таблица используется для агрегированной статистики.

### `user_word_progress`

Персональная статистика пользователя по отдельным словам.

| Поле | Описание |
| --- | --- |
| `user_id` | ссылка на пользователя |
| `word_id` | ссылка на слово |
| `correct_count` | число правильных ответов |
| `wrong_count` | число ошибок |
| `last_result` | последний сохраненный результат |

На этой таблице построены:

- weak words;
- mistakes quiz;
- topic progress.

### `daily_practices`

Сохраненные сессии daily practice.

| Поле | Описание |
| --- | --- |
| `user_id` | ссылка на пользователя |
| `practice_date` | дата сессии |
| `word_ids_json` | сериализованный набор слов на день |
| `current_index` | индекс текущего вопроса |
| `correct_answers` | число правильных ответов |
| `is_completed` | флаг завершения |
| `completed_at` | время завершения |

Эта таблица делает daily practice стабильной в течение дня и позволяет восстановить ту же сессию при повторном входе.

## Связи

```text
users 1 --- N training_attempts
users 1 --- N user_word_progress
users 1 --- N daily_practices

word_sets 1 --- N words
word_sets 1 --- N training_attempts

words 1 --- N user_word_progress
```

## Регистрация на уровне базы

При первой регистрации пользователя:

1. бот получает `telegram_id`, `username` и `full_name` из Telegram update;
2. пользователь проходит шаги онбординга;
3. выбранные значения языка и уровня сохраняются в `users`;
4. поле `bilingual_ui` получает значение по умолчанию;
5. при следующих заходах поиск идет по `telegram_id`.

Таким образом, запись в `users` является основной точкой привязки всего пользовательского состояния.

## Замечания

- профиль пользователя отделен от статистики обучения;
- общая история попыток отделена от progress по отдельным словам;
- daily practice хранится в отдельной таблице, а не только во временном состоянии.
