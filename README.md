# EngyBot

Telegram-бот для изучения языка. Telegram здесь используется как клиентский интерфейс, а основная логика работает на бэкенде: профиль пользователя, подбор контента, квизы, повторение ошибок, прогресс и дневная практика.

## Текущее состояние

В репозитории находится рабочий MVP со следующими возможностями:

- онбординг и постоянный профиль пользователя;
- контент по уровням `A1-C2`;
- словарные темы и grammar units;
- короткие квиз-сессии с мгновенной обратной связью;
- повторение на основе ошибок пользователя;
- дневная практика, закрепленная за текущей датой;
- прогресс и базовая статистика;
- опциональный bilingual UI.

## Основной сценарий работы

1. Пользователь отправляет `/start`.
2. Бот проверяет, существует ли профиль в базе.
3. Если профиля нет, запускается онбординг:
   - родной язык;
   - изучаемый язык;
   - текущий уровень.
4. Если профиль уже существует, пользователь сразу попадает в главное меню.
5. Дальше бот открывает один из основных разделов:
   - `Course`
   - `Practice`
   - `Review`
   - `Profile`
   - `Progress`
   - `Help`
6. Результаты сохраняются в PostgreSQL и используются в повторении, статистике и прогрессе.

## Регистрация и профиль

Под регистрацией в этом проекте понимается не создание логина и пароля, а первичное заполнение учебного профиля внутри Telegram-бота.

Что происходит технически:

1. После `/start` бот ищет пользователя по `telegram_id`.
2. Если запись не найдена, запускается FSM-сценарий онбординга.
3. Пользователь последовательно выбирает:
   - `source_language`
   - `target_language`
   - `level`
4. После завершения шагов бот создает запись в таблице `users`.
5. В профиль также сохраняются:
   - `username`
   - `full_name`
   - `bilingual_ui`
6. После этого бот использует профиль для:
   - фильтрации словаря и grammar units;
   - настройки bilingual UI;
   - выбора тем и сложности учебных сессий.

Регистрация не повторяется на каждом запуске. При следующем `/start` бот находит запись в `users` и сразу открывает главное меню.

## Возможности

### Course

- словарные темы;
- grammar units по CEFR;
- мини-диалоги с пошаговой навигацией;
- двуязычные подписи для целевого и родного языка.

### Practice

- короткие квиз-сессии;
- несколько форматов квизов;
- мгновенная обратная связь после каждого ответа;
- дневная практика с фиксированным набором на текущий день.

### Review

- список слабых слов;
- квиз по ошибкам;
- персистентный прогресс по словам.

### Progress

- завершенные попытки квизов;
- точность;
- прогресс по темам;
- сводка по цели дня.

## Быстрый запуск

### Docker

```bash
cp .env.example .env
docker compose up --build
```

Перед запуском нужно указать `BOT_TOKEN` в `.env`.

### Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
cp .env.example .env
python -m bot
```

Если бот запускается без Docker, `DATABASE_URL` должен указывать на локальный PostgreSQL, а не на хост `db`.
